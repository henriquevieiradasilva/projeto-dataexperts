## Projeto Vendas

### Objetivo do Projeto

Este projeto tem como objetivo a construção de uma **arquitetura completa de Data Lakehouse**, utilizando o **padrão Medallion** (Bronze → Silver → Gold), aplicada a um cenário de vendas.

A solução contempla desde a **ingestão de dados** brutos até a disponibilização de dados analíticos prontos para consumo, com **implementações visuais** no Power BI para geração de insights de negócio.

<br>

### Arquitetura de dados

A arquitetura segue o padrão Medallion, com as seguintes camadas:

- **Source**: Dados totalmente brutos, em seus formatos originais (CSV).

- **Bronze**: Dados ainda brutos, porém padronizados em formato estruturado (ex.: Delta/Parquet).

- **Silver**: Dados limpos, tratados, tipados corretamente e com regras de qualidade aplicadas.

- **Gold**: Dados consolidados, modelados (fato e dimensões) e prontos para análise e visualização.

<br>

### Escopo da Solução

1. Ingestão e tratamento de dados de vendas
2. Implementação da arquitetura Lakehouse
3. Modelagem dimensional (Fato e Dimensões)
4. Criação de KPIs e métricas de negócio
5. Desenvolvimento de dashboards analíticos no Power BI
6. Geração de análises e insights estratégicos

<br>

### Tecnologias Utilizadas

- Databricks / Apache Spark
- Delta Lake / Parquet
- Python / PySpark
- Power BI
- Git & GitHub
